# monitoring edu

nload برای مانیتورینگ ترافیک شبکه در لینوکس کاملاً مناسبِ و جوابِ خوبی‌ست. نمای گرافیکی ساده (خطی) از ورودی/خروجیِ هر اینترفِیس می‌دهد و مجموع، نرخ لحظه‌ای و میانگین را نمایش می‌دهد.

چند دستور و مثال سریع:

نصب (Debian/Ubuntu):
```
sudo apt update
sudo apt install nload
```

اجرای nload برای یک اینترفیس مشخص (مثلاً eth0):
```
sudo nload eth0
```

اجرای nload بدون مشخص کردن اینترفیس → لیست و امکان انتخاب اینترفیس‌ها را نشان می‌دهد:
```
sudo nload
```

خروج: q

نکات کوتاه:

nload برای مانیتورینگ زنده و سریع خیلی خوبه، مخصوصاً وقتی می‌خواهی ببینی چه مقدار دانلود/آپلود در هر لحظه داره مصرف می‌شه.

nload گزارش بلندمدت ذخیره نمی‌کنه؛ برای مانیتورینگ تاریخی یا گزارش‌گیری از vnstat یا ابزارهای جمع‌آوری لاگ مثل Prometheus + node_exporter بهتره.

چند جایگزین مفید (هر کدوم مزیت خودش رو داره):

iftop — نمایش اتصالات فعال و پهنای‌باند مصرفی به تفکیک host/port.

nethogs — نشان می‌دهد هر پروسه چقدر پهنای‌باند مصرف می‌کند.

bmon — نمای گرافیکی و آمار لحظه‌ای چند اینترفیس.

vnstat — مناسب برای ثبت طولانی‌مدت و آمار روزانه/ماهیانه.

tcpdump — برای تحلیل بسته‌ای و عمیق (نه برای overview پهنای‌باند).
